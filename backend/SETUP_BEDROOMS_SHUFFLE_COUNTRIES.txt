================================================================================
HOW PROPERTIES ARE SHUFFLED
================================================================================

The shuffle happens ONCE when the page first loads:
1. User visits http://localhost:3000
2. Properties load from backend
3. Frontend shuffles them randomly using Fisher-Yates algorithm
4. Properties display in new random order
5. User navigates through pages 1-7
6. Order stays the same until they refresh the page
7. Refresh page → properties shuffle again

Why shuffle?
- Makes the website feel fresh on each visit
- Shows different featured properties without changing database
- Better user experience for browsing

================================================================================
IF SOMETHING GOES WRONG
================================================================================

Problem: Bedrooms/bathrooms still showing as dashes
Solution:
1. Make sure backend is running (python main.py)
2. Run the script again: python assign_bedrooms_bathrooms.py
3. Hard refresh browser: Ctrl+F5

Problem: New properties not showing
Solution:
1. Verify script ran successfully: "33 properties added"
2. Hard refresh browser: Ctrl+F5
3. Check pagination shows 7 pages (82 ÷ 12 = 7 pages)
4. Scroll to page 7 to see newest properties

Problem: Properties always in same order
Solution:
1. This is normal - refresh the page to reshuffle
2. Each page refresh shows new random order
3. Order only stays same within a single browsing session

Problem: Backend crashes
Solution:
1. Stop backend: Press Ctrl+C in terminal
2. Restart backend: python main.py
3. Try again
