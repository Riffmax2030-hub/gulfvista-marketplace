PREVENTION - INDEX DEFINITION RULES
================================================================================

When adding new models or columns, follow these rules:

RULE 1: unique=True automatically creates an index
   ✅ email = Column(String(255), unique=True)
   ❌ email = Column(String(255), unique=True, index=True)
   ❌ With Index("ix_email", "email") in __table_args__

RULE 2: index=True creates an index
   ✅ status = Column(String(50), index=True)
   ❌ status = Column(String(50), index=True, index=True)
   ❌ Plus Index("ix_status", "status") in __table_args__

RULE 3: __table_args__ Index() for composite/custom indices
   ✅ __table_args__ = (Index("ix_composite", "field1", "field2"),)
   ❌ Don't duplicate indices defined at column level

RULE 4: Check before adding
   Before adding an index, check:
   1. Does the column have unique=True? (creates index automatically)
   2. Does the column have index=True? (already creates index)
   3. Is there already an Index() for this field in __table_args__?

   If ANY of these are true, don't add another index definition!

FUTURE DEVELOPMENT CHECKLIST
================================================================================

When adding new tables/columns to models.py:

□ Define the table and columns
□ Check: Does any column have unique=True? (Don't add index=True)
□ Check: Does any column have index=True? (Don't add Index() in __table_args__)
□ Add __table_args__ ONLY for composite indices or when index=True is NOT used
□ Add protective comments explaining why indices are defined that way
□ Test with: python verify_database_tables.py
□ Test with: python test_backend_quick.py
□ Check server logs for any "DuplicateTable" or "DuplicateIndex" errors
□ If errors occur, review index definitions against the rules above
